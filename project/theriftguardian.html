The Rift Guardian

The Rift Guardian is a wave-based survival shooter VR game with a space theme, where you are dropped off in the middle of nowhere in outer space tasked to survive as long as possible against large hordes of aliens and robots out for your blood. You can use a wide range of power-ups and weapons to give yourself the upper hand in battle. Use machine guns, bombs, or shields to protect yourself. Can you make it out alive?

A little about the project:

This project was meant to be THE assignment of the semester where we had a 6 month timeframe to learn how Unreal Engine works and develop a VR game inside of it. We had to do so within an agile environment and this time in an actual office space so that we truly got to feel what it’s like developing a project like a regular studio would do and as an actual dev team with an actual name in a real office environment hosted by a company called XR Lab. The team’s name is Devil’s Luck and this is the only game we made under this name.

My role within the project

I was tasked to make a healing item and a flashlight that stuns enemies. The healing item was the most difficult out of the two because the healing item had to represent a syringe both aesthetically and mechanically. The player would have to stab themselves with the syringe to heal up so I had to figure out a way to check collisions and then apply the heal (for which the system has already been setup thanks to my wonderful colleague Gino).

Image of the syringe in the editor



Next up we’ve got the memory eraser which is inspired by the Neuralyzer from the Men In Black movies. The way that it works is you charge it by holding down either triggers and then releasing to activate a flash that would stun all the enemies that stared at it. It worked by setting the custom time dilation of the enemy component (basically how fast time passes for that component) to 0 so the enemy completely halts any and all movement and processing essentially freezing it in place.



Unfortunately I wasn’t able to get any footage of the flash in action due to the repository containing all the stuff I need being private and I unfortunately no longer have access to it because Devil’s Luck is no longer operational within XR Lab’s office space.

The struggles

The only thing that I really struggled with during the development of this project was the fact that this was a completely new engine that I was tasked to pick up and develop a feature within. I have however worked with Unreal Engine for Fortnite so I was able to navigate myself within the engine, I just wasn’t familiar with the concept of Blueprints nor did I have any knowledge about C++. I also haven’t ever worked with VR beforehand so after this I can say I’m at least familiar with the technical side of VR.

Aside from being completely unfamiliar with how Unreal Engine works I’ve truly had no bigger struggle so far than I’ve had with input. This is because the way that input is registered within Unreal Engine is with Input Mapping Contexts.

An Input Mapping Context is essentially a component that you assign inputs to (basically input mappings for either keyboards, mice or controllers.) so that the game actually recognizes and takes input from said buttons. Without this, if you were to assign an action to a button without this it just wouldn’t work because by default Unreal Engine doesn’t know what to do when you press a button. Any other engine recognizes a button press by default and all you really have to do is listen to it. In Unreal Engine you have to go out of your way to define said buttons so that they can then be listened to.

What did I learn from all of this?

This project was more of a learning experience rather than a project with a fully polished product as the end goal because it was our first time using Unreal Engine and none of us had ever touched the engine and haven’t had any experience with it beforehand. I did however pick up alot during these 6 months. I’ve learned alot about the ins and outs of Unreal Engine and I’ve also familiarized myself with VR a little so I could potentially provide support for VR in my future projects.
